# Spatio-Temporal Partitioning Demo


I think NAV2 in jazzy should work. Although if we use the simple_nav api I rewrote 
the nav2 simple nav for rolling and may be incompatible with this.

This demo contains the client code for the Spatio-Temporal allocation. To get 
started you will need to run the rest API that actually allocates the spaces from here:
https://github.com/arjo129/mapf_post/blob/main/examples/rest_api.rs

You can do this by checking out the mapf_post library:

```
git clone git@github.com:arjo129/mapf_post.git
```  

And then running the cargo command:

```
cargo run --example rest_api -p <trajectory.csv>
``` 
Trajectory.csv is generated by a MAPF planner like: https://github.com/open-rmf/mapf or https://github.com/arjo129/pibt_rs
Or for smaller test cases you could look at https://github.com/arjo129/mapf_post/tree/main/example_trajectories

To start the simulation run:
```
ros2 launch sp_demo_nav2_bringup cloned_multi_tb3_simulation_launch.py robots:="robot0={x: 0.0, y: 5.0, yaw: 0.0}; robot1={x: 3.0, y: 5.0, yaw: 0.0};"	
```
The poses in the simulation should correspond to the poses in  your `trajectory.csv` files.

Personally, Id start the simulation, before launching the rest server to prevent funny behaviours
